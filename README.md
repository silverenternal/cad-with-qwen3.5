# CAD 图纸识别 - 智能对话助手

基于 Ollama 多模态大模型的 CAD 图纸智能分析工具，支持本地和云端部署。通过自然语言对话与 AI 交互，实现图纸理解、数据提取和专业咨询。

## ✨ 核心特性

- 🖼️ **多模态支持** - 同时处理图像和文本，精准识别 CAD 图纸
- 💬 **交互式对话** - 多轮对话保留上下文，支持连续追问
- 🔒 **安全设计** - 防止程序自动退出，支持优雅中断 (Ctrl+C)
- 📦 **灵活部署** - 支持 Ollama Cloud 和本地模型
- 🔐 **密钥管理** - .env 文件安全存储 API Key
- 📸 **动态图片加载** - 通过 `@路径` 方式随时附加图片
- 🗃️ **图片缓存** - 自动缓存已加载图片，避免重复加载

## 🚀 快速开始

### 1. 环境要求

- **操作系统**: Windows / Linux / macOS
- **依赖工具**: `curl` (通常系统已预装)
- **Rust 环境**: 用于编译源码

### 2. 安装步骤

```bash
# 克隆或进入项目目录
cd vl_tes

# 编译 Release 版本
cargo build --release

# 运行程序
cargo run --release
```

### 3. 配置 API Key（可选）

如果需要使用 Ollama Cloud，配置 API Key：

**方法一：使用 .env 文件（推荐）**
```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env 文件，填入你的 API Key
# 从 https://ollama.com/connect 获取
```

**方法二：运行时输入**
- 首次运行时按提示输入
- 可选择保存到 .env 文件

**方法三：环境变量**
```bash
# Windows (PowerShell)
$env:OLLAMA_API_KEY="your-api-key"

# Linux/Mac
export OLLAMA_API_KEY="your-api-key"
```

## ⚙️ 高级配置

### config.toml 配置文件

创建 `config.toml` 自定义默认设置：

```toml
# 默认图片路径（可选）
default_image_path = ""

# 本地模型
default_local_model = "llava:7b"

# 云端模型
default_cloud_model = "qwen3.5:397b-cloud"
```

### 模型选择

**云端模型（推荐）：**
- `qwen3.5:397b-cloud` - 通义千问，强大的多模态能力
- 需要有效的 API Key
- 无需本地部署

**本地模型：**
- `llava:7b` - 开源多模态模型
- 需要运行 Ollama 服务：`ollama serve`
- 拉取模型：`ollama pull llava:7b`

## 🔧 故障排除

### 常见问题

#### 1. curl 命令未找到
```bash
# Windows: 通常已预装
# Linux: sudo apt install curl
# macOS: brew install curl
```

#### 2. API Key 无效
- 检查 .env 文件格式是否正确
- 确认 API Key 未过期
- 删除 .env 文件重新输入

#### 3. 模型不存在
```bash
# 云端模型：检查模型名称是否正确
# 本地模型：ollama pull <model_name>
```

#### 4. 图片加载失败
- 检查图片路径是否正确
- 确认图片格式支持（JPG/PNG）
- 检查文件权限

#### 5. 程序卡住或无响应
- 按 `Ctrl+C` 优雅退出
- 检查网络连接（云端模式）
- 确认 Ollama 服务运行（本地模式）

### 日志说明

程序输出带时间戳的日志：
```
[19:30:15] ✅ 已加载图片：xxx.jpg
[19:30:16] ℹ️  正在思考...
[19:30:20] 🤖 AI 响应...
```

## 📝 技术栈

- **语言**: Rust 2021 Edition
- **异步运行时**: Tokio
- **HTTP 客户端**: curl (命令行)
- **序列化**: Serde + Serde JSON
- **加密编码**: Base64
- **配置管理**: TOML + dotenv
- **中断处理**: ctrlc

## 🔐 安全注意事项

1. **不要提交敏感信息**
   - `.env` 已加入 `.gitignore`
   - 不要手动提交包含真实 API Key 的文件

2. **保护导出结果**
   - 识别结果可能包含敏感信息
   - 妥善保管导出的文件

3. **本地模型安全**
   - 确保 Ollama 服务仅本地访问
   - 不要暴露到公网
